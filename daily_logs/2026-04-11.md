### The shuffle is the centerpiece
Wrote a 3-pass constraint shuffle (`pipeline/shuffle.py`):
1. **Generate** — repeat clips to fill target duration, randomize
2. **Repair** — scan violations, swap offenders with safe candidates further down the list
3. **Validate** — second scan must return zero or the shuffle raises `ShuffleValidationError` and the orchestrator refuses to render

**Cross-validation results:**
- **Our new shuffle on the 127 real Hawaii clips produces 0 violations at comfort rule (N=8, S=4)** (1 repair pass)
- **Hawaii production reference had 782 violations** under the same rule
- Target rule (N=8, S=8) is mathematically infeasible with 127 clips × 19 reps; documented in config with empirical notes. S=4 is the comfort floor we can actually hit.
- All 10 shuffle unit tests pass (including synthetic tiny projects, standalone-clip edge cases, Hawaii filename parser)

### Key architectural decisions
- **Content root is separate from code root.** `Desktop\Still Frequency\` stays the source of truth (Levi already set it up); `C:\StillFrequency\` is just code. Config-driven paths, no hardcoding.
- **Honor the existing intake contract.** Slug-as-DSL (`432hz_midnight_pass`), optional `notes.txt`/`title.txt`/`description.txt`/`thumbnail_frame.jpg`/`thumbnail_notes.txt`, lifecycle markers (`READY`/`RUNNING`/`DONE`/`FAILED`). No new `config.json` schema.
- **Split render pipeline** — video-only NVENC on Predator via SSH, separate audio build on Dell, mux with `-c:v copy` (no re-encode), ffprobe validate, then thumbnail + metadata + upload.
- **Fragmented MP4 safety** only in the mux step (per `mux_audio_final.bat`); video-only render uses `+faststart` per the proven `render_528hz_hawaii_remote.ps1`.
- **SaaS-ready from day 1** — Supabase schema has `user_id` on every row, RLS policies ready; Next.js dashboard built with shadcn aesthetic; Stripe/auth deferred until signal exists.
- **Project-sourced thumbnails are enforced in code** — `thumbnail.py` raises `WrongSourceError` if the hero frame isn't inside the project folder. Locks in Levi's rule after the past wrong-source incident.
